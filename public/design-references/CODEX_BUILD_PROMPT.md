# 给 Codex 的建站提示词

你是一个资深全栈工程师和产品设计师。请帮我从零搭建一个可上线的个人主页网站，网站名称是「贵宾小吴系统」。我会把 8 张设计参考图放在 public/design-references/ 目录下，请严格参考这些图的视觉风格：明亮、阳光、夏天、果味、成熟清爽、奶油白/浅黄/橙黄/青绿/湖蓝配色，圆角卡片、轻阴影、中文界面、桌面端优先并兼容移动端。

重要：这些参考图是视觉参考，不要把图片直接当网页截图铺上去。请用真实 HTML/CSS/React 组件实现网站。

## 技术栈
请使用：
- Next.js App Router + TypeScript
- Tailwind CSS
- Supabase：数据库、Auth、Storage
- Vercel 部署兼容
- 表单校验使用 zod / react-hook-form 或等价方案
- 图片上传使用 Supabase Storage

## 角色和权限
网站有两种模式：
1. 公开访问者模式：任何人可以访问主页和公开内容，只能查看六个模块，不允许看到「编辑、更新、上传、添加、删除、保存」等管理按钮。
2. 作者管理模式：我登录 /admin 后可以编辑所有内容、上传图片/PDF、管理留言、发布动态、归档内容、修改站点设置。

请实现安全的权限判断，不要只靠前端隐藏按钮。后端 API、数据库 RLS 和上传权限也要限制。

## 公开首页
首页标题：欢迎来到贵宾小吴系统
首页以 3 行 x 2 列排列六个大矩形模块卡片，任何地方可点击进入。六个模块：
1. 关于我
2. 我的项目
3. 我的作品
4. 实习经历
5. 证书/奖项
6. 我的简历

首页底部背景使用 public/images/home-faint-background-source.jpg 作为极淡、低透明度、去饱和的底部水印背景。不要让它喧宾夺主。

## 六个模块
### 1. 关于我
公开页展示：自我介绍、喜欢的人或事、GitHub、邮箱、小红书、微信二维码。
管理页可编辑：自我介绍、喜欢的人或事标签、GitHub账号、邮箱、微信二维码图片上传、小红书账号。

### 2. 我的项目
公开页展示：项目名称、研究方向/项目状态、项目背景、核心目标、项目地址/研究地址、更新时间、备注。
管理页可新增、编辑、删除项目，可上传项目封面图。

### 3. 我的作品
公开页展示：作品名称/App名称/网址名称、作品地址、简介、标签、配图。页面内容可无限延展，下拉查看更多。
管理页可新增、编辑、删除作品，可为每个作品上传多张配图。

### 4. 实习经历
公开页展示：公司/组织、岗位、时间、主要职责、关键成果、相关链接。
管理页可新增、编辑、删除实习经历。

### 5. 证书/奖项
公开页展示：证书/奖项名称、类型、时间、颁发机构、证书图片。
管理页可通过加号添加新证书/奖项，可编辑、删除、设置是否公开。

### 6. 我的简历
公开访问时必须先输入访问码。通过后只能查看或下载我上传的简历图片/PDF。
管理页可上传、替换、删除简历图片和 PDF。
访问码校验必须在服务端完成，不要把访问码明文写在前端。

## 动态、归档、留言板、设置
顶部导航保留：主页、动态、归档、留言板、设置。
- 动态：作者可发布短动态，访客可查看公开动态。
- 归档：按月份或类型自动整理公开作品、项目、动态。
- 留言板：访客可留言，但留言默认不公开。只有作者和留言者本人可以看到该留言及作者回复。留言提交后生成一个私密查看链接或查询码，留言者凭该链接/查询码查看作者回复。作者后台可以查看、回复、删除留言，并可手动批准某条留言公开展示。
- 设置：作者可改站点标题、副标题、主题色、简历访问码等。

## 数据库设计
请设计并创建 Supabase SQL migration，至少包括：profile、projects、works、work_images、internships、certificates、resume_files、posts、guest_messages、site_settings。启用 RLS。公开内容只读；管理操作只允许作者账户；留言默认 private/pending。

## 交互要求
- 公开页和管理页风格统一，但公开页隐藏所有管理按钮。
- 所有上传都要有拖拽上传区域和文件大小/格式提示。
- 表单保存后有 Toast 提示。
- 删除前有确认弹窗。
- 空状态要友好，不要出现假内容冒充真实经历。
- 允许我先用占位内容，之后后台编辑。

## 安全要求
- 简历文件放在私有 bucket，通过访问码验证后再生成临时 signed URL。
- 管理后台需要登录。
- 管理员邮箱通过环境变量 ADMIN_EMAIL 限制。
- 不要提交 .env.local 到 git。
- 请提供 .env.example。

## 执行方式
请先给我实施计划和文件结构，然后开始创建项目。每完成一阶段就运行检查：npm run lint、npm run build。如果缺少环境变量，请使用 .env.example 并在 README 写清楚我需要填什么。
